# Daria VPN Bot

ربات فروشگاه VPN مبتنی بر Hiddify با سیستم فروشنده، مشتری و سوپرادمین.

## راه‌اندازی

### پیش‌نیازها

```bash
pip install -r requirements.txt
```

### تنظیمات `.env`

| متغیر | توضیح |
|--------|--------|
| `TOKEN` | توکن ربات از BotFather |
| `BOT_ID` | یوزرنیم ربات |
| `BOT_LINK` | لینک ربات |
| `ADMIN_LIST` | آیدی‌های سوپرادمین (با کاما) |
| `MATIN` | آیدی دریافت خطاها |
| `DATABASE` | مسیر فایل SQLite |
| `HIDDIFY_*` | تنظیمات پنل Hiddify |

### اجرا با tmux

```bash
tmux new-session -d -s dariavpnbot -c "/root/VPN Bots/dariavpnbot"
tmux send-keys -t dariavpnbot 'source venv/bin/activate && python3 main.py' Enter
```

مشاهده لاگ:

```bash
tmux attach -t dariavpnbot
```

### اجرا با systemd + tmux (سرور)

پروژه روی سرور در مسیر `/root/VPN Bots/dariavpnbot` نصب شده است.

```bash
# نصب وابستگی‌ها (یک‌بار)
cd "/root/VPN Bots/dariavpnbot"
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt

# کپی تنظیمات
cp .env.example .env   # سپس مقادیر را پر کنید

# مدیریت سرویس
systemctl start dariavpnbot    # استارت ربات در tmux
systemctl stop dariavpnbot     # توقف
systemctl status dariavpnbot   # وضعیت
systemctl enable dariavpnbot   # اجرای خودکار بعد از ریبوت
```

سرویس systemd: `dariavpnbot.service` — ربات را داخل سشن tmux با نام `dariavpnbot` اجرا می‌کند.

## سیستم ترافیک فروشنده

### سوپرادمین

- **📦 مدیریت بسته‌های حجمی**: تعریف بسته ترافیک با قیمت تومان و دلار
- **💳 مدیریت ولت‌ها**: ولت رمزارز + کارت بانکی برای دریافت پرداخت ترافیک
- **🛍 مدیریت فروشندگان**: تخصیص حجم، toggle نمایش کارت به هر فروشنده
- **👥 گروه پرداخت فروشندگان**: تنظیم گروه تأیید فیش برای هر فروشنده یا گروه پیش‌فرض
- **💳 نمایش کارت فروشندگان**: روشن/خاموش کردن نمایش شماره کارت مدیریت هنگام خرید ترافیک هر فروشنده
- تأیید دستی فیش خرید ترافیک فروشندگان

### فروشنده

- **🔄 تمدید / خرید ترافیک**: مشاهده بسته‌ها، پرداخت، ارسال فیش
- **📦 مدیریت بسته‌های من**: ایجاد، ویرایش (نام/حجم/مدت/قیمت) و حذف بسته‌های فروش با UI اینلاین
- **📊 آمار فروش و درآمد**: نمایش کل درآمد، تعداد فروش، تفکیک نوع تراکنش و جزئیات بسته‌ها
- **➕ ساخت کانفیگ دستی**: ساخت VPN با حجم و روز دلخواه (کسر از ترافیک فروشنده)
- **📋 کانفیگ‌های دستی من**: مشاهده کانفیگ‌های ساخته‌شده
- هشدار خودکار وقتی ترافیک زیر ۲۰ گیگابایت باشد

### مشتری

- اگر فروشنده ترافیک کافی نداشته باشد، خرید مسدود و پیام «سرویس در حال آپدیت» نمایش داده می‌شود
- فروشنده از تلاش خرید ناموفق مطلع می‌شود
- فیش پرداخت به گروه اختصاصی فروشنده (یا گروه پیش‌فرض) ارسال می‌شود

## گروه پرداخت فروشندگان

سوپرادمین از منوی **👥 گروه پرداخت فروشندگان** می‌تواند:

1. **لیست گروه‌ها** — مشاهده گروه هر فروشنده
2. **تنظیم گروه برای فروشنده** — با فوروارد پیام از گروه یا وارد کردن آیدی عددی
3. **گروه پیش‌فرض** — برای فروشندگانی که گروه اختصاصی ندارند
4. **اعمال پیش‌فرض** — اختصاص گروه پیش‌فرض به همه فروشندگان بدون گروه

> ربات باید در گروه پرداخت **ادمین** باشد تا بتواند فیش مشتریان را ارسال کند.

کلید `DEFAULT_SELLER_PAYMENT_GROUP` در جدول `bot_settings` ذخیره می‌شود.

## نمایش کارت به فروشندگان

از منوی **💳 نمایش کارت فروشندگان** سوپرادمین می‌تواند:

1. **لیست فروشندگان** — مشاهده وضعیت هر فروشنده (✅ روشن / ❌ خاموش) و تغییر با یک کلیک
2. **روشن/خاموش برای همه** — اعمال یکجا روی تمام فروشندگان

ستون `show_card_for_traffic` در `seller_configs` این تنظیم را ذخیره می‌کند.

## دیتابیس و migration

تمام جداول و ستون‌ها در `traffic_service.run_migrations()` متمرکز مدیریت می‌شوند. در هر استارت ربات جداول ساخته می‌شوند، ستون‌های جدید به DB قدیمی اضافه می‌شوند و `verify_schema()` صحت را بررسی می‌کند.

جداول: `users`, `seller_configs`, `packages`, `receipts`, `invite_links`, `admin_list`, `channels`, `block_list`, `uploaded_files_new`, `bot_settings`, `seller_packages`, `admin_wallets`, `charge_doc_channel`
